## PAGE 5: Chapter 1 - Introduction 

 

### CHAPTER 1: INTRODUCTION 

 

#### 1.1 Overview 

The **Property Rental & Flat Management System** is a web-based application designed to facilitate the buying, selling, and renting of flats and apartments. In the modern digital era, people prefer searching for property online rather than visiting physical locations. This system bridges the gap between Property Owners and Potential Clients (Tenants/Buyers). 

 

The application is built using the MERN stack logic (React, Node.js, MySQL). It provides a user-friendly interface where clients can browse available flats and owners can manage their listings efficiently. 

 

#### 1.2 Problem Statement 

Currently, finding a suitable flat for rent or purchase requires a lot of manual effort. Clients have to rely on brokers or newspaper advertisements which are often outdated. Property owners struggle to update the status of their flats (e.g., Sold, Booked) in real-time. There is no central platform for scheduling appointments or checking availability instantly. 

 

#### 1.3 Objectives 

The main objectives of this project are: 

1.  To create a central platform for Property Rental and Management. 

2.  To allow Clients to browse flats and check availability status (Available/Sold). 

3.  To enable Property Owners to register, login, and manage their flat listings. 

4.  To implement an Appointment Scheduling system for clients. 

5.  To maintain a secure database of flats and user details using MySQL.

## PAGE 7: Chapter 2 - System Analysis 

 

### CHAPTER 2: SYSTEM ANALYSIS 

 

#### 2.1 Existing System 

In the existing manual system, property management is done using paper records or simple spreadsheets. 

*   **Disadvantages:** 

    *   Data redundancy and duplication. 

    *   Difficult to update flat status immediately. 

    *   Clients cannot book appointments online. 

    *   High risk of data loss (fire, water damage to paper). 

 

#### 2.2 Proposed System 

The proposed "Property Rental & Flat Management System" is a digital solution that automates the process. 

*   **Advantages:** 

    *   **Centralized Database:** All data is stored securely in MySQL. 

    *   **Real-time Updates:** Owners can change flat status instantly. 

    *   **Accessibility:** Users can access the system from anywhere via the internet. 

    *   **Security:** User accounts are protected by login authentication.

## PAGE 14: Chapter 5 - Implementation 

 

### CHAPTER 5: IMPLEMENTATION DETAILS 

 

#### 5.1 Front-End Implementation (React & Bootstrap) 

React.js is used to create interactive user interfaces. We used **Functional Components** and **React Hooks** (like `useState`, `useEffect`) to manage the state of the application. 

 

*   **Component Structure:** 

    *   `Header.js`: Contains Navigation Bar. 

    *   `FlatCard.js`: Reusable component to display individual flat details. 

    *   `Dashboard.js`: Private route for Property Owners. 

 

*   **Bootstrap Integration:** 

    We used Bootstrap classes like `container`, `row`, `col-md-4`, `btn-primary` to style the buttons and forms quickly and responsively. 

 

#### 5.2 Back-End Implementation (Node.js & Express) 

Node.js acts as the server. Express.js is the framework used to handle routing. 

 

*   **API Routes:** 

    *   `POST /api/auth/register`: To register a new Property Owner. 

    *   `POST /api/auth/login`: To authenticate the user. 

    *   `GET /api/flats`: To fetch all flats for the homepage. 

    *   `PUT /api/flats/:id`: To update flat details or status.

## PAGE 17: Features Implementation 

 

#### 5.4 Feature-wise Implementation 

 

**A. Homepage:** 

The Homepage uses the `useEffect` hook to call the backend API (`/flats`) as soon as the page loads. It maps through the JSON data and displays it using the `FlatCard` component. 

 

**B. Appointment Scheduling:** 

A form is provided on the Flat Detail page. When a user fills in their Name, Email, and Message, a `POST` request is sent to the database, saving the appointment details for the owner to view. 

 

**C. Flat Status Update:** 

In the Owner Dashboard, there is a dropdown menu for each flat. The Owner selects the status (Sold/Booked). On clicking "Update", the `PUT` API is called, updating the specific row in the MySQL database.
